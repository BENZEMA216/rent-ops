# 模式：verify — 假房源检测

输入房源链接，对该房源执行多维度真实性校验，综合通用检测 + 平台专项信号，输出风险等级和处置建议。

## 输入

- **完整模式**：用户输入 `/rent verify {url}`
- **Lite 模式**：被 auto-evaluate 或 scan 内部调用，自动评估后附带真实性校验

## 模式选择

| 模式 | 触发方式 | 输出详细度 |
|------|----------|------------|
| 完整模式 | 用户主动调用 `/rent verify {url}` | 分节报告，每条信号都有说明 |
| Lite 模式 | auto-evaluate / scan 内部调用 | 单行摘要 + 综合等级 |

## 平台识别

根据 URL 域名判断平台，选择对应的专项检测规则：

| URL 域名 | 平台 | 专项检测 |
|----------|------|----------|
| `douban.com` | 豆瓣 | 发帖人主页、评论区揭露、文案模板 |
| `xiaohongshu.com` / `xhs.link` | 小红书 | 评论区翻车、博主主页、图片风险 |
| `ke.com` / `lianjia.com` | 贝壳/链家 | 经纪人挂房量、房源状态 |
| `58.com` / `anjuke.com` | 58同城/安居客 | 同图检测、刷新频率、认证状态 |
| `ziroom.com` | 自如 | 用户差评集中度 |

如果 URL 不匹配任何已知平台，仅执行通用检测层，并在报告中注明「未识别平台，跳过专项检测」。

## 风险等级定义

| 等级 | 标记 | 含义 |
|------|------|------|
| 高风险 | 🔴 RED | 有明确的欺骗或诈骗信号，建议直接放弃 |
| 中风险 | 🟡 YELLOW | 存在可疑点，需谨慎核实后再行动 |
| 低风险 | 🟢 GREEN | 未发现明显异常，可以正常推进 |

## 综合判定规则

- **1 条及以上 RED 信号** → 综合等级：高风险（硬拦截，输出强警告）
- **2 条及以上 YELLOW 信号（无 RED）** → 综合等级：中风险（软警告，建议核实）
- **其他情况** → 综合等级：低风险

## 通用检测层

以下 5 项检测对所有平台执行。

### 1. 价格异常检测

**触发方式：** 对比月租与同区域同户型均价。

- WebSearch `"{小区名} 租金" OR "{商圈} {户型} 租金均价"` 获取参考价
- 如果 `${CLAUDE_SKILL_DIR}/data/listings.json` 中有同区域历史房源，可用作内部参考价

**判定规则（当前价格）：**

| 偏差 | 等级 |
|------|------|
| 低于均价 30% 及以上 | 🔴 RED |
| 低于均价 15%–30% | 🟡 YELLOW |
| 在均价 ±15% 以内 | 🟢 GREEN |

**判定规则（历史价格波动，如有历史数据）：**

如果 `${CLAUDE_SKILL_DIR}/data/listings.json` 中该房源有历史挂牌价记录：

| 波动情况 | 等级 |
|----------|------|
| 7 天内降价 30% 及以上 | 🔴 RED |
| 14 天内降价 15%–30% | 🟡 YELLOW |

规则标识符：`price_below_avg`、`price_drop_7d`、`price_drop_14d`

---

### 2. 隔断检测

**触发方式：** 对比户型与面积。

| 情况 | 等级 |
|------|------|
| 一居室面积 < 15㎡ | 🔴 RED |
| 两居室面积 ≤ 30㎡ | 🔴 RED |
| 一居室面积 15–20㎡ | 🟡 YELLOW |

规则标识符：`partition_1br_tiny`、`partition_2br_tiny`、`partition_1br_small`

---

### 3. 信息一致性检测

**触发方式：** 对比标题、正文、联系方式中的关键字段，检查是否自相矛盾或信息严重缺失。

| 情况 | 等级 |
|------|------|
| 标题与正文直接矛盾（如标题写「整租」，正文写「合租」；标题写「南向」，正文写「无窗」） | 🔴 RED |
| 关键信息含糊或缺失（如面积、租金、地址均未提及，或使用「价格面议」） | 🟡 YELLOW |

规则标识符：`info_contradiction`、`info_vague`

---

### 4. 联系方式风险检测

**触发方式：** 分析房源页面中联系方式的要求和形式。

| 情况 | 等级 |
|------|------|
| 要求先转账/先付押金才能看房；或完全没有联系方式 | 🔴 RED |
| 仅提供微信，拒绝电话或平台内沟通 | 🟡 YELLOW |

规则标识符：`contact_transfer_required`、`contact_missing`、`contact_wechat_only`

---

### 5. 挂牌时长检测

**触发方式：** 读取房源的发布时间或首次抓取日期，计算挂牌天数。

| 挂牌时长 | 等级 |
|----------|------|
| > 90 天 | 🔴 RED |
| 45–90 天 | 🟡 YELLOW |
| < 45 天 | 🟢 GREEN |

规则标识符：`listing_age_90d`、`listing_age_45d`

---

## 平台专项检测层

### 豆瓣（douban.com）

#### 发帖人主页分析

打开发帖人豆瓣主页，检查：

| 情况 | 等级 |
|------|------|
| 注册时间 < 7 天，且发帖记录全为房源帖；或同时在多个不同城市发布租房帖 | 🔴 RED |
| 注册时间 < 30 天 | 🟡 YELLOW |

规则标识符：`douban_poster_new_spam`、`douban_poster_multi_city`、`douban_poster_new`

#### 评论区揭露

读取房源帖评论，检查是否有租客留言投诉：

| 情况 | 等级 |
|------|------|
| 3 条及以上评论出现「中介」「假房源」「骗子」等关键词 | 🔴 RED |
| 1–2 条评论出现上述关键词 | 🟡 YELLOW |

规则标识符：`douban_comments_fraud`、`douban_comments_warn`

#### 文案模板检测

分析正文是否全为模板文字、无具体信息：

| 情况 | 等级 |
|------|------|
| 全篇为通用模板文字，无任何具体描述（楼层、朝向、小区名、照片等均无） | 🔴 RED |
| 部分使用模板，但缺少关键具体信息 | 🟡 YELLOW |

规则标识符：`douban_full_template`、`douban_partial_template`

---

### 小红书（xiaohongshu.com）

#### 评论区翻车

读取笔记评论区，检查是否有用户反映实地不符：

| 情况 | 等级 |
|------|------|
| 3 条及以上评论出现「去了不是这套」「和图片不符」「实地差很多」等 | 🔴 RED |
| 1–2 条评论出现上述描述 | 🟡 YELLOW |

规则标识符：`xhs_comments_mismatch`、`xhs_comments_warn`

#### 博主主页分析

打开博主主页，分析近 30 篇内容：

| 情况 | 等级 |
|------|------|
| 近 30 篇全为房源招租帖，无其他生活内容 | 🔴 RED |
| 近 30 篇中 70% 以上为房源招租帖 | 🟡 YELLOW |

规则标识符：`xhs_profile_all_listings`、`xhs_profile_mostly_listings`

#### 图片风险

检查配图来源和质量：

| 情况 | 等级 |
|------|------|
| 图片带有其他平台水印（如贝壳、安居客），或存在明显专业后期修图痕迹 | 🔴 RED |
| 全为官方样板间照片，无任何实拍图或生活细节照 | 🟡 YELLOW |

规则标识符：`xhs_image_watermark`、`xhs_image_no_real_shots`

---

### 贝壳/链家（ke.com / lianjia.com）

#### 经纪人挂房量

查看该经纪人名下当前在售/在租房源数量：

| 情况 | 等级 |
|------|------|
| 挂房量 50 套及以上 | 🔴 RED |
| 挂房量 30–50 套 | 🟡 YELLOW |

规则标识符：`ke_agent_listings_50`、`ke_agent_listings_30`

#### 房源状态异常

检查房源页面状态：

| 情况 | 等级 |
|------|------|
| 房源页面返回 404 或显示「已下架」，但仍可通过搜索检索到 | 🔴 RED |
| 房源超过 60 天未更新任何信息（如照片、描述、价格） | 🟡 YELLOW |

规则标识符：`ke_listing_404`、`ke_listing_stale`

---

### 58同城/安居客（58.com / anjuke.com）

#### 同图检测

反向搜索房源主图，检查是否被多个帖子使用：

| 情况 | 等级 |
|------|------|
| 同一图片出现在 3 个及以上不同房源帖中 | 🔴 RED |
| 同一图片出现在 2 个房源帖中 | 🟡 YELLOW |

规则标识符：`58_image_reuse_3`、`58_image_reuse_2`

#### 刷新频率

检查房源是否频繁刷新挂牌时间：

| 情况 | 等级 |
|------|------|
| 每天刷新一次（近 7 天连续刷新） | 🔴 RED |
| 每 2–3 天刷新一次 | 🟡 YELLOW |

规则标识符：`58_refresh_daily`、`58_refresh_frequent`

#### 认证状态

检查发布者是否通过平台实名/机构认证：

| 情况 | 等级 |
|------|------|
| 未认证 + 价格明显低于市场均价 | 🔴 RED |
| 未认证（价格正常） | 🟡 YELLOW |

规则标识符：`58_uncertified_low_price`、`58_uncertified`

---

### 自如（ziroom.com）

#### 用户差评集中度

搜索或读取该房源/该房间的历史入住评价：

| 情况 | 等级 |
|------|------|
| 3 条及以上用户投诉（如「实际房况与描述严重不符」「隐藏费用」「无法退押金」） | 🔴 RED |
| 1–2 条用户投诉 | 🟡 YELLOW |

规则标识符：`ziroom_complaints_3`、`ziroom_complaints_1`

## 输出格式

### 完整模式输出

```
🔍 假房源检测报告：{小区名 / 标题}
━━━━━━━━━━━━━━━━━━━━━━━━━━

📋 通用检测

  价格异常：{GREEN/YELLOW/RED} — {说明，如「月租 2800，低于均价 35%」}
  隔断检测：{等级} — {说明}
  信息一致性：{等级} — {说明}
  联系方式：{等级} — {说明}
  挂牌时长：{等级} — {说明，如「已挂牌 112 天」}

🏠 平台专项检测（{平台名}）

  {检测项}：{等级} — {说明}
  {检测项}：{等级} — {说明}
  ...

📊 综合判定：{高风险 / 中风险 / 低风险}

💡 综合建议：
  {高风险} → 建议放弃。检测到明确欺骗信号：{列出 RED 信号}。继续联系风险极高。
  {中风险} → 谨慎推进。发现 {N} 个可疑点，建议核实后再联系：{列出 YELLOW 信号}。
  {低风险} → 可正常推进。未发现明显真实性问题。
```

### Lite 模式输出（单行）

```
{高风险} → ⛔ 疑似假房源，建议放弃。是否仍要查看完整报告？
{中风险} → ⚠️ 存在 {N} 个可疑点，建议核实。
{低风险} → ✅ 真实性校验通过。
```

## 数据更新规则

完整模式检测完成后，将结果写入 `${CLAUDE_SKILL_DIR}/data/listings.json` 对应房源条目的 `verify` 字段：

```json
{
  "verify": {
    "level": "high | medium | low",
    "date": "YYYY-MM-DD",
    "signals": [
      {
        "rule": "{规则标识符}",
        "level": "RED | YELLOW | GREEN",
        "detail": "{说明文字}"
      }
    ]
  }
}
```

如果 `${CLAUDE_SKILL_DIR}/data/listings.json` 中尚无该房源条目，创建新条目并包含 `verify` 字段。
