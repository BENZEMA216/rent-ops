# 模式：visit — 看房准备

从 tracker 中选择要看的房源，生成看房路线、现场 checklist 和砍价话术。

## 输入

用户输入 `/rent visit`，agent 展示当前状态为 `已评估` 或 `约看中` 的房源列表，让用户选择本次要看的几套。

> "你 tracker 里有以下待看房源：
>
> #001 南山-荣上居 1室1厅 4500元 ⭐4.2
> #003 福田-xxx花园 2室1厅 5200元 ⭐4.0
> #005 宝安-xxx小区 1室1厅 3200元 ⭐3.5
>
> 你这次打算看哪几套？（输入编号，如 1,3）"

## 生成内容

### 1. 看房路线

读取选中房源的小区名和区域信息，规划最优看房顺序：

- WebSearch `"{小区A} 到 {小区B} 地铁"` 查询各点之间交通
- 按地理位置排序，减少折返
- 估算总耗时

输出：

```
🗺️ 看房路线（{N} 套，预计 {X} 小时）

1️⃣ {小区A}（{区域}）
   └→ 地铁约 {X}min → 2️⃣

2️⃣ {小区B}（{区域}）
   └→ 地铁约 {X}min → 3️⃣

3️⃣ {小区C}（{区域}）

建议从 {起点} 出发，先看 {理由}。
```

### 2. 每套房的现场 Checklist

为每套房生成定制化 checklist，根据该房源的评估报告中发现的薄弱维度增加重点检查项。

**通用 checklist（所有房源都检查）：**

```
📋 看房 Checklist：{小区名} {户型}

基础设施
- [ ] 水压：厨房和卫生间同时开水龙头
- [ ] 热水：热水器类型？多久出热水？
- [ ] 电路：插座数量够不够？有无跳闸隐患？
- [ ] 网络：是否有宽带接口？测速
- [ ] 燃气：是否通燃气？灶具能用？
- [ ] 空调：制冷制热是否正常？

居住环境
- [ ] 采光：几点有阳光？对面楼遮挡？
- [ ] 通风：开窗后空气流通？
- [ ] 隔音：敲墙听厚度、走廊噪音、窗外噪音
- [ ] 潮湿：墙面有无霉斑？衣柜内部？
- [ ] 门锁：是否智能锁？能否换锁？

家具家电
- [ ] 逐项核对清单：{从房源描述中提取}
- [ ] 有无损坏？
- [ ] 缺什么需要自己买？

小区和周边
- [ ] 实际走一遍到最近地铁口
- [ ] 超市/便利店在哪？
- [ ] 楼道环境、电梯状况
- [ ] 邻居素质（观察楼道堆物、噪音）
- [ ] 垃圾处理（有无集中投放点？气味？）
```

**定制检查项：** 如果评估报告中某维度分数低，增加重点检查：
- 安全分低 → 增加：检查门禁是否形同虚设、快递柜是否在小区内
- 风险分低 → 增加：对照避雷帖中的问题逐项验证
- 房况分低 → 增加：重点检查装修细节、老化程度

### 3. 砍价话术

为每套房生成砍价参考：

**信息收集：**
- WebSearch `"{小区名}" 租金 最新` 查同小区近期成交价
- 估算挂牌时长（如果有发布时间信息）
- 当前租房市场冷热度（WebSearch `"{城市}" 租房市场 {月份}`）

**话术模板：**

```
💰 砍价参考：{小区名}

市场参考：
- 同小区近期成交价：{范围}
- 挂牌时长：约 {X} 天（越久砍价空间越大）
- 当前市场：{淡季/旺季}

建议出价：{金额}元（在挂牌价基础上降 {X}%）

话术：
1️⃣ 首次报价：
   "这套房我看了挺喜欢的，不过我也在看同小区/附近的几套。
    价格上您能优惠到 {目标价} 吗？我可以马上定。"

2️⃣ 如果对方不让步：
   "我了解到同小区最近有 {参考价} 成交的，您看 {折中价} 可以吗？
    我可以接受押一付三。"

3️⃣ 签约前最终谈判：
   "如果 {最终价} 可以的话，我今天就签合同。
    另外，{免费期/维修/配置} 方面能不能再谈一下？"
```
